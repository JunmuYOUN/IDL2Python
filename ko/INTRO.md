👋 **IDL → Python 변환+실행검증(Parity) 하네스**입니다.

IDL(`.pro`)을 Python으로 변환한 뒤, **원본 IDL을 서버에서 실제로 실행해 만든 산출물(오라클)**과
변환 Python의 산출물을 체크포인트 단위로 수치 대조하고, 일치할 때까지 수정 루프를 돕니다.

---

### 📋 입력 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| **① IDL 파일** | `.pro` 파일/디렉토리/Git/웹 URL | `/path/to/your_routine.pro` |
| **② 변환 목적** | 왜 변환하는지 | `SunPy 파이프라인 통합` |
| **③ 작업 경로** | 결과물 저장 디렉토리 | `/path/to/work/your_routine_parity/` |
| **④ 검증 데이터 방식** | 어떻게 확보할지 | `직접 제공` / `필요 스펙 요청` / `자동 다운로드` / `서버 내 탐색` |
| **⑤ 오라클 범위** | IDL 실행 대조 범위 | `전체 프로그램 1회` / `특정 함수 단위` |

### 💬 프롬프트 템플릿

```
[IDL 파일]을 Python으로 변환하고 IDL 실행 결과와 대조 검증해줘.

파일: [경로/URL]
목적: [목적]
작업 경로: [/저장/경로]
검증 데이터: [직접 제공 경로 | 스펙 요청 | 자동 다운로드 | 서버 내 탐색]
```

---

### 🔬 핵심 특징

- 기대값을 추정하지 않습니다 — **IDL을 실제로 돌려서** 만든 값과 대조합니다.
- 최종 출력만이 아니라 **중간 체크포인트 전부**를 대조해, 불일치가 나면
  **어느 블록에서 처음 벌어졌는지** 자동으로 짚어줍니다 (전치/픽셀 시프트/스케일 자동 진단 포함).
- 허용 오차·배열 축 방향·마스크 IoU 기준은 `policy.yaml`에 선언되고 비교기가 기계적으로 판정합니다.
- 수정 루프에서 IDL은 다시 돌리지 않습니다 (오라클 캐시).

### ✅ 진행 중 확인받는 것

1. **G1**: 변환 계획 + 체크포인트 계획 + 비교 정책 + 데이터 확보 방식
2. **G2**: 외부 데이터 다운로드 직전 (출처/용량)
3. **G3**: 최종 결과 + parity certificate

준비되시면 템플릿대로 입력해 주세요. 🚀
